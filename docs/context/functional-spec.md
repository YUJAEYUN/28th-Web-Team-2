# 또래 인식 서비스 — 기능 명세서

> **범위**: 프론트엔드 / 백엔드 구현 대상 기능만 기술합니다.
> AI 형용사 추출·칸 배정 알고리즘·이미지 생성은 별도 AI 파이프라인 문서에서 다룹니다.

---

## 1. 서비스 개요

친구들의 설문 응답을 바탕으로 "나를 찍은 인생네컷(4컷 이미지 + 설명문)"을 만들어주는 서비스.
결과는 **조하리의 창** 4칸 구조로 표현됩니다.

| 칸 | 의미 |
|----|------|
| ① 공개된 나 | 나도 알고, 친구도 아는 모습 |
| ② 친구만 아는 나 | 나는 모르는데, 친구는 아는 모습 |
| ③ 나만 아는 나 | 나는 아는데, 친구는 모르는 모습 |
| ④ 새롭게 발견될 나 | 데이터가 부족해 아직 발견되지 않은 모습 |

---

## 2. 사용자 역할

| 역할 | 정의 |
|------|------|
| **생성자** | 서비스를 시작하고 자기 설문을 푼 뒤 링크를 공유하는 사람 (인생네컷의 주인공) |
| **참여자** | 생성자의 링크를 통해 접속해 생성자에 대한 설문을 푸는 사람 |

---

## 3. 인증 및 식별

- **비회원 서비스** — 로그인 없음
- 프론트엔드는 사용자가 처음 웹에 진입할 때 **UUID를 생성해 localStorage에 저장**
- 이 ID가 설문 응답의 참여자 식별자로 사용됨
- localStorage를 삭제하면 식별 불가 (의도적 우회는 허용 범위로 간주)

---

## 4. 핵심 비즈니스 규칙

| 규칙 | 내용 |
|------|------|
| 문항 수 | 생성자 자기 설문 **8개**, 참여자 설문 **8개** (각각 독립 추출) |
| 문항 출처 | 4개 성향(개방성·성실성·외향성·우호성) × 50문항 = 총 200개 풀에서 랜덤 추출 |
| 생성자·참여자 문항 동일 불필요 | 서로 다른 문항을 풀어도 됨 |
| 결과 생성 시점 | **세션 생성 시각으로부터 24시간 후** 고정 (응답 완료 시점 기준이 아님) |
| 결과 생성 조건 | 24시간 내 참여자 **3명 이상** 응답 완료 |
| 조건 미달 시 처리 | 결과를 생성하지 않음. 생성자는 **새 세션을 시작**해야 함 |
| 결과 접근 권한 | **링크 보유자 누구나** 결과 조회 가능 |
| 중복 참여 | 동일 localStorage ID의 재응답 허용 (중복 방어는 FE에서 처리) |
| 링크 유효 기간 | 없음 (영구) |
| 결과 생성 횟수 | 제한 없음 (새 세션을 열어 다시 진행 가능) |

## 5. 사용자 플로우

### 5-1. 생성자 플로우

```
[서비스 진입]
    ↓
[온보딩] — 서비스 설명
    ↓
[자기 설문] — 200개 풀에서 랜덤 8문항 출제
    - 각 문항: 5지선다 상황극
    - 본인 설문에만 "잘 모르겠다" 6번째 선택지 존재
    ↓
[설문 완료] — 세션 생성 (24시간 타이머 시작)
    ↓
[링크 생성 & 공유] — 참여자에게 공유할 고유 링크 발급
    ↓
[대기 화면] — 남은 시간 / 현재 응답자 수 표시
              (페이지 진입 및 새로고침 시 API 호출로 업데이트)
    ↓
[24시간 경과]
    ├── 응답자 3명 이상 → [결과 화면]
    └── 응답자 3명 미만 → [실패 안내] → [새 세션 시작] → 온보딩으로 이동
```

### 5-2. 참여자 플로우

```
[생성자 링크 접속]
    ↓
[간단 안내] — "누구에 대한 설문인지" 표시
    ↓
[설문] — 200개 풀에서 랜덤 8문항 출제
    - 각 문항: 5지선다 상황극 ("나" → "이 사람"으로 주어 치환)
    - 중립 선택지 없음
    ↓
[응답 완료] — 완료 메시지 표시
```

---

## 6. 설문 문항 규칙

| 항목 | 내용 |
|------|------|
| 총 문항 풀 | 200개 (성향 4개 × 50문항) |
| 출제 수 | 세션당 8개 랜덤 추출 |
| 형태 | 5지선다 상황극 (동의 척도·리커트 척도 없음) |
| 주어 | 생성자 설문은 "나", 참여자 설문은 "이 사람" |
| 중립 선택지 | 생성자 설문에만 6번째 선택지("잘 모르겠다") 존재 / 참여자 설문에는 없음 |
| 성향 태그 | 각 문항에 성향 태그 있으나 화면에 미노출 (내재화) |

---

## 7. 세션 & 링크

| 항목 | 내용 |
|------|------|
| 세션 생성 시점 | 생성자가 자기 설문을 완료한 순간 |
| 세션 ID | 서버에서 발급하는 고유 ID |
| 링크 형태 | `{서비스_도메인}/survey/{세션_ID}` |
| 링크 유효 기간 | 없음 (영구) |
| 참여 가능 기간 | 세션 생성 후 24시간 이내 |
| 결과 조회 | 24시간 경과 후 조건 충족 시 링크로 영구 접근 가능 |

---

## 8. 결과 화면

### 8-1. 정상 결과

- 링크 보유자 누구나 결과 화면 접근 가능
- **4칸 모두 AI가 생성한 이미지 1컷 + 설명문** 표시 (빈 칸 없음)
- 이미지와 설명문은 AI 파이프라인이 생성 (구현 대상 아님)
- FE는 API 응답으로 받은 이미지 URL과 텍스트를 렌더링

### 8-2. 결과 생성 실패 (응답자 3명 미만)

| 항목 | 내용 |
|------|------|
| 화면 | 결과 없음 안내 |
| 안내 문구 | "24시간 내에 친구 3명의 응답을 받지 못했어요. 새로 시작해볼까요?" |
| 액션 | "새로 시작하기" → 온보딩으로 이동 (새 세션 생성) |
