# 또래 인식 서비스 — 결과 산출 알고리즘 기획서

> 친구들의 설문으로 "나를 찍은 네 장의 사진(인생네컷)"을 만들어주는 서비스.
> 이 문서는 결과가 어떤 원리로 만들어지는지 정리한 기획서입니다.

---

## 1. 용어 정리

### Big Five (빅파이브)

사람의 성격을 5가지 성향으로 보는 심리학 측정 틀. **이 서비스는 신경성을 제외한 4개만 사용합니다.**

| 성향 | 쉽게 말하면 |
|------|------------|
| 개방성 | 새로운 것·모험을 좋아하나 |
| 성실성 | 책임감 있고 꼼꼼한가 |
| 외향성 | 사람들과 어울리는 걸 좋아하나 |
| 우호성 | 남에게 친절하고 잘 맞춰주나 |

> **Big Five는 "내재화"만 합니다.** 문항 설계 시 "이 질문이 어떤 성향을 건드리나"를 잡아주는 숨은 잣대로만 쓰입니다. 서비스 화면이나 AI 프롬프트에는 Big Five라는 말도, 성향 점수도 전혀 나오지 않습니다.

### 조하리의 창

"나에 대한 정보를 누가 알고 있는가"를 네 칸으로 나눈 틀.

| 칸 | 뜻 |
|----|-----|
| ① 공개된 나 | 나도 알고, 친구도 아는 모습 |
| ② 친구만 아는 나 | 나는 모르는데, 친구는 아는 모습 |
| ③ 나만 아는 나 | 나는 아는데, 친구는 모르는 모습 |
| ④ 새롭게 발견될 나 | 나도 친구도 확실히 모르는 모습 |

> **Big Five는 "내용"(이 사람이 어떤 사람인지)을 보고, 조하리는 "인식"(그 모습을 누가 알고 있는지)을 나눕니다.**

---

## 2. 두 이론이 맞물리는 방식

- Big Five → 뒤에서 문항을 설계합니다(내재화). "이 질문은 개방성을 건드리는 상황극이다"
- AI → 문항의 답변을 읽고 형용사를 뽑아냅니다. "이 사람, 대담하고 즉흥적이네"
- 조하리 → 그 형용사를 어느 칸에 넣을지 정합니다. "근데 그걸 친구만 알아"

**점수를 매기지 않습니다.** 설문 답을 숫자로 환산하지 않고, AI가 답변을 그대로 읽어 형용사로 옮깁니다.

```
본인이 푼 답   → AI → 본인이 본 나의 형용사 집합
친구들이 푼 답 → AI → 친구들이 본 나의 형용사 집합
두 집합이 겹치는지, 한쪽에만 있는지 → 조하리 칸 배정
```

## 3. 전체 플로우

```
[설문]          →  [AI 형용사 추출]    →  [형용사 집합]    →  [조하리 칸 배정]   →  [산출물]
본인+친구가         각 답변을 AI가          유저별로             본인 집합 vs         에이전트가
5지선다             상황·행동·특질로         형용사 모음           친구 집합 비교        이미지 생성
상황극 응답         읽고 형용사 후보 산출     (본인/친구)           (겹침·차이)          + 설명문
```

## 4. 단계별 설명

### 4-1. 설문

- **200개 문항 풀**: 성향 4개 × 성향당 50개
- **형태**: 무조건 5지선다 상황극 (동의 척도 없음, 모든 문항이 구체적 상황)
- **출제**: 사람마다 랜덤 **8개** 추출. 매번 다른 질문
- **본인·참여자 문항 불일치 허용**: 서로 다른 문제를 풀어도 됨 (점수가 아니라 형용사를 뽑는 방식이라 무관)
- **성향 태그 숨김**: 각 문항에 "어느 성향을 재는가" 태그가 있지만 화면에는 미노출

**역할별 설문**

| 역할 | 행동 |
|------|------|
| 생성자(주인공) | 링크 만들기 전 온보딩에서 자기 설문 필수 선행 |
| 참여자 | 다른 사람의 링크로 들어가 그 사람에 대한 설문 응답 |

- 비회원 서비스라 **재설문 없음** (본인 설문은 온보딩 1회)
- 본인 설문에 **"잘 모르겠다" 중립 선택지** 존재 (본인이 자신을 잘 모르는 것이 서비스의 연료 — 흐릿할수록 ②④의 반전이 생김)
- 세션 생성 후 **24시간 내 친구 3명 응답** 필요

### 4-2. AI 형용사 추출

모든 답변을 AI에게 넘기고 유저별로 형용사를 추출:

```json
{
  "raw_response": "식당에서 메뉴판을 펼쳤다. 이 사람의 선택은 안 먹어본 거 한번 도전했다",
  "candidate_johari_adjectives": [
    "Searching (탐구심 있는)",
    "Bold (대담한)",
    "Spontaneous (자발적/즉흥적인)",
    "Independent (독립적인)"
  ]
}
```

결과로 유저별 두 묶음:
- **본인이 본 나**: 주인공이 자기 설문에서 만든 형용사 집합
- **친구들이 본 나**: 참여자들이 만든 형용사 집합

### 4-3. 형용사 집합 → 조하리 칸 배정 (핵심 로직)

| 칸 | 조건 | 예시 |
|----|------|------|
| ① 공개된 나 | 본인 집합 ∩ 친구 집합 (둘 다 자주 언급) | 본인도 "활발한", 친구도 "활발한" |
| ② 친구만 아는 나 | 친구 집합에만 강하게 등장 | 본인은 안 썼는데 친구들이 "사실은 모험가" |
| ③ 나만 아는 나 | 본인 집합에만 등장 | 본인은 "다정한"이라는데 친구는 모름 |
| ④ 새롭게 발견될 나 | 양쪽 다 약하거나 흩어짐 | 어느 집합에서도 또렷하게 안 잡힘 |

### 4-4. 형용사 가이드라인

형용사 표현 유형:
- 단일형: "다정한", "조용한"
- 조합형: "조용히 챙겨주는", "겉은 무뚝뚝한데 속은 따뜻한"
- 반전형: "스스로는 신중하다지만 사실은 모험가"

| 가이드라인 | 내용 |
|-----------|------|
| 허용 톤 | 긍정적이거나 중립적인 표현만 |
| 금지 표현 | 비하·외모 평가·차별적 단어, 상처 줄 수 있는 직설적 부정어 |
| 부정 신호 처리 | 약점으로 보일 결과는 부드럽거나 매력적인 표현으로 환원 (예: 내향성 → "혼자만의 시간을 아는") |
| 다양성 vs 일관성 | 핵심 형용사는 고정, 곁들이는 표현만 다양하게 |

### 4-5. 산출물

**인생네컷 (이미지)**
- **4칸 모두 AI로 생성**
- 에이전트가 각 칸의 형용사를 그림 주제로 삼아 분위기에 맞는 컷을 생성

**설명문 (글)**
- AI가 본인·친구 형용사와 근거를 읽어 문장 생성
- 예: "넌 스스로 신중하다 했지만 친구들은 모험가로 봤어"

---

## 5. 한 줄 요약

> 설문(본인+친구, 랜덤 5지선다) → AI가 답변을 형용사로 추출 → 유저별 형용사 집합 → 본인·친구 집합 비교로 네 칸 배정 → 칸마다 형용사로 인생네컷과 설명문 완성.

---

## 6. 결정 현황

1. **응답 방식** — 무조건 5지선다 상황극. 동의 척도 없음.
2. **문항 수** — 성향 4개 × 50 = 총 200개 풀. 사람마다 랜덤 **8개** 출제.
3. **본인·참여자 문항 불일치 허용** — 서로 다른 문제를 풀어도 됨.
4. **Big Five 내재화** — 점수 산출 없음. 문항 설계용 숨은 잣대로만 사용.
5. **생성자 본인 설문 선행** — 링크 만들기 전 온보딩에서 본인 설문 필수.
6. **재설문 제거** — 비회원 서비스라 본인 설문은 온보딩 1회.
7. **산출물 4칸 모두 AI 생성** — 빈 칸 없음.

---

## 7. 문항 설계 규칙

- **구조**: 성향 4개(개방성·성실성·외향성·우호성) × 성향당 50문항 = 200개
- **형태**: 전 문항 5지선다 상황극
- **주어**: "이 사람" = 친구가 풀 땐 그대로, 본인이 풀 땐 "나"로 치환
- **출제**: 200개 풀에서 사람마다 랜덤 **8개** 추출. 본인·참여자가 같은 문항일 필요 없음
- **성향 태그**: 각 문항에 "어느 성향을 건드리는가" 태그가 숨어 있음 (화면·프롬프트 미노출)
- **선택지 설계**: 다섯 보기는 한 행동 스펙트럼의 양 끝과 중간을 덮도록 구성
